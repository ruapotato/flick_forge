Tips for Good Requests

✓ Do

  • Be specific about features you need
  • Describe the problem the app should solve
  • Reference existing Flick apps for style inspiration
  • Keep it touch-friendly and mobile-first
  • Keep scope reasonable for AI generation

✗ Don't

  • Request GTK, terminal, or non-QML apps
  • Request clones of complex commercial apps
  • Ask for apps requiring proprietary APIs
  • Request malicious or harmful software
  • Be too vague ("make a cool app")

Note: All apps are built as Qt/QML for Flick mobile shell. They should match the style of existing apps like Music Player, Calculator, and Notes.
